Le moteur de templates par défaut de Django est appelé Django Template Language (DTL). Il permet de séparer la logique de présentation de la logique de contrôle dans les applications web. Voici une description du fonctionnement de DTL ainsi que des éléments clés tels que les blocs, les boucles for, les structures de contrôle if et l'affichage de variables passées par contexte.

  1. Fonctionnement du moteur de templates : DTL fonctionne en remplaçant les balises de template dans un fichier de template par des valeurs dynamiques fournies par une vue. Les balises de template sont des marqueurs spéciaux dans le fichier de template qui indiquent à DTL comment générer le contenu dynamique. Les balises de template sont entourées de doubles accolades, par exemple {{ variable }}.
  2. Blocs : Les blocs sont utilisés pour définir des sections de contenu dans un template qui peuvent être remplacées par du contenu spécifique à une vue particulière. Les blocs sont définis à l'aide de la balise {% block %} et peuvent contenir du contenu par défaut qui sera utilisé si aucun contenu spécifique n'est fourni. Les blocs sont utiles pour définir des sections de contenu communes à plusieurs pages, telles que l'en-tête et le pied de page.
  3. Boucles for ... in : Les boucles for sont utilisées pour itérer sur une liste d'éléments et afficher chacun d'entre eux dans le template. Les boucles for sont définies à l'aide de la balise {% for %} et peuvent contenir des balises de template pour afficher les éléments de la liste. Les boucles for sont utiles pour afficher des listes d'éléments tels que des articles de blog ou des produits dans une boutique en ligne.
  4. Structures de contrôle if : Les structures de contrôle if sont utilisées pour afficher du contenu conditionnel dans un template en fonction de la valeur d'une variable. Les structures de contrôle if sont définies à l'aide de la balise {% if %} et peuvent contenir des balises de template pour afficher le contenu conditionnel. Les structures de contrôle if sont utiles pour afficher du contenu spécifique en fonction de l'état d'une application, tel que l'affichage d'un message d'erreur si une connexion échoue.
  5. Affichage de variables passées par contexte : Les variables peuvent être passées à un template à l'aide d'un contexte. Le contexte est un dictionnaire Python qui contient des paires clé-valeur, où les clés sont les noms des variables et les valeurs sont les valeurs des variables. Les variables peuvent être affichées dans un template à l'aide de balises de template telles que {{ variable }}. Les variables sont utiles pour afficher du contenu dynamique dans un template, tel que le nom d'un utilisateur connecté.

En résumé, DTL permet de séparer la logique de présentation de la logique de contrôle dans les applications web en utilisant des balises de template pour générer du contenu dynamique. Les blocs, les boucles for, les structures de contrôle if et l'affichage de variables passées par contexte sont des éléments clés de DTL qui permettent de créer des templates flexibles et réutilisables.
